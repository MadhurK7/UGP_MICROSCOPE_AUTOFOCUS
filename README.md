# Research-Grade Autofocus for Colloidal Microscopy

Closed-loop, illumination-aware autofocus system designed for
OpenFlexure-style microscopes running on a Raspberry Pi.
Combines a sweep-based coarse search with an adaptive fine controller,
multi-metric scoring, and confidence-gated control.

## Quick start

### Install

```bash
pip install opencv-python numpy scipy matplotlib pyserial
```

### Run the validation suite (no hardware needed)

```bash
python scripts/run_validation.py
```
This builds a synthetic z-stack, evaluates every metric, runs the
closed-loop on a simulated stage with backlash, and writes plots to
`validation_out/`.

### Run on real hardware

```bash
python scripts/run_live.py --port /dev/ttyACM0 --camera 0 --coarse
```

`--coarse` runs the OpenFlexure-style sweep before fine tracking.
Without it, the system jumps straight into fine tracking from the
current z. `--simulate` runs the same code path against the synthetic
rig used in tests.

### Run the test suite

```bash
python tests/test_integration.py
python tests/test_coarse.py
python tests/test_end_to_end.py
```

## Module-by-module summary

### Part 1 — `autofocus/preprocessor.py`
Eight-stage pipeline: grayscale → illumination flat-field → saturation
mask → conditional CLAHE → mild denoise → particle ROIs → centre crop
→ scene-condition classification. Every stage exists for a
microscopy-specific failure mode documented inline.

### Part 2 — `autofocus/metrics.py`
Five metric primitives, each with a documented strength and failure
mode:

| Metric | Strength | Failure mode |
|---|---|---|
| Tenengrad | textured fields | Airy-halo bias |
| Laplacian | fast | sensitive to noise |
| Brenner | sharp peak | direction-biased |
| JPEG | brightness-decoupled | encode cost |
| Counting | scale-invariant | pre-smooth required |

`AdaptiveCombiner` picks weights from a lookup table keyed by the
preprocessor's scene-condition string. `ConfidenceEstimator` produces
the `[0,1]` confidence the fine controller uses to suppress motion.

### Part 3 — `autofocus/coarse.py`
Symmetric sweep over a configurable z range:
1. Overshoot the start by `backlash_steps` in the opposite direction
   of approach.
2. Approach the start, then sample N points monotonically.
3. After the sweep, identify the discrete argmax.
4. Refine sub-step with a 3-point parabolic fit.
5. Park the stage at the discrete max from below (one-direction
   motion the entire way).
6. Quality-gate the result on SNR + prominence; fail the sweep loudly
   if the curve was too flat to be trusted.

### Part 4 — `autofocus/fine.py`
State machine: `INIT → CLIMB → REFINE → LOCKED` with a `HOLD` branch
for low-confidence frames.

- **Deadband** suppresses motion when |Δscore| is within the noise
  floor.
- **Hysteresis** requires N consecutive regression frames before
  flipping direction.
- **Adaptive step size** halves on each plateau.
- **Cooldown** after a reversal lets backlash settle before more
  decisions are made.
- **Rolling-median smoothing** rejects single-frame outliers.
- **Drift detection** re-engages CLIMB if the LOCKED score falls a
  configurable fraction below the all-time best.

### Part 5 — `autofocus/system.py` + `stage_iface.py`
Top-level `AutofocusSystem` owns one of every component. Two methods:
- `coarse_focus()`: runs the sweep once, parks the stage, resets
  the fine controller's bookkeeping.
- `track(...)`: per-frame closed loop with optional callback for
  display / external logging. Records every frame as a `FrameRecord`
  for offline analysis.

`SerialStage` talks the OpenFlexure firmware protocol
(`mr <dx> <dy> <dz>`, `p?`, `release`, `dt`, `ramp_time`). `NullStage`
is a drop-in replacement for tests / dry runs.

### Part 6 — `validation/validator.py`
Scientific tests:
1. **Per-metric accuracy**: `pred_peak` from each metric vs ground
   truth; reports MAE, SNR, prominence.
2. **Repeatability**: predicted peak across N noisy trials.
3. **Timing**: mean and p95 ms per pipeline stage.
4. **Oscillation**: counts direction reversals + RMS drift in a
   tracking trace.

Plots: focus curves, peak-error bar, timing bars, tracking trace.

## Tuning

| Parameter | When to change | Effect |
|---|---|---|
| `coarse_range_steps` | DoF / mechanical play | wider sweep covers more ground but takes longer |
| `coarse_n_samples` | curve smoothness | more samples → finer peak resolution |
| `coarse_backlash` | observed mechanical play | must exceed actual backlash |
| `fine.coarse_step` | tracking responsiveness | bigger = faster but oscillates more |
| `fine.deadband` | noise floor | bigger = more stable, less responsive |
| `fine.hysteresis_frames` | noisy frames | more = more stable |
| `fine.stable_required` | lock speed | bigger = slower lock, more conservative |
| `confidence_floor` | trust threshold | higher = motor stops more often |

## Raspberry Pi performance

On a Pi 4 (cool CPU, 64-bit OS, OpenCV 4.x):

| Stage | Mean (ms) | p95 (ms) |
|---|---|---|
| preproc | 5–7 | 8 |
| metrics | 3–4 | 5 |
| combine + conf | < 1 | 1 |
| **full pipeline** | **~10** | **~14** |

This leaves comfortable headroom inside the 33 ms budget at 30 FPS.

## Hardware integration

The system speaks the OpenFlexure motor-board firmware protocol
unchanged. It works with:
- Sangaboard v2 / v3
- Stock Arduino Nano + ULN2003 + 28BYJ-48 stepper running
  `OpenFlexure_Nano_Motor_Board.ino`
- Any firmware that accepts `mr <x> <y> <z>\n` and replies `done.\n`

The `SerialStage` requires only `pyserial`. No labthings dependency,
no LabThings server required to use this library standalone.
